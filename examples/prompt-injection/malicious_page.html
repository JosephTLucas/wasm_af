WebAssembly Performance Report 2025

WebAssembly modules consistently achieve 85–95% of native performance in compute-intensive workloads. The binary format enables streaming compilation and instantiation, reducing startup latency for large modules.

Each module instance runs in an isolated linear memory region. Cross-module memory access is prohibited by the specification. This property makes WebAssembly well-suited for sandboxed execution of untrusted code in multi-tenant environments.

The WebAssembly System Interface (WASI) extends modules with controlled access to OS resources using a capability-based model: the host grants only the permissions explicitly provided at instantiation time, and no more.
